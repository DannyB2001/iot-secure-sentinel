# Raspberry Pi Gateway Setup - Iris Gateway / IoT Secure Sentinel

## Overview

This Raspberry Pi acts as an IoT gateway between HARDWARIO IoT nodes and the cloud backend application.

Gateway responsibilities:

- Receive telemetry from HARDWARIO nodes
- Communicate via MQTT
- Process and transform sensor payloads in Node-RED
- Detect alarms and offline states
- Downsample telemetry data
- Store queued events locally
- Forward processed events to the backend API
- Survive reboot and power loss automatically

## Local Persistence

### Goal

Prevent data loss during:

- Backend outage
- Network outage
- Raspberry Pi reboot

### Persistence Strategy

Node-RED uses:

```text
contextStorage = localfilesystem
```

Configured in:

```text
~/.node-red/settings.js
```

Configuration:

```javascript
contextStorage: {
  default: {
    module: "localfilesystem"
  }
},
```

This ensures:

- Queue survives reboot
- Queue survives Node-RED restart
- Buffered telemetry is resent automatically

## Outbox Queue Architecture

```text
Incoming event
    ->
Add to local queue
    ->
Periodic sender
    ->
HTTP POST
    ->
Success -> remove from queue
Failure -> keep in queue
```

## Automatic Startup After Reboot

### Enabled Services

Node-RED:

```bash
sudo systemctl enable nodered.service
```

Mosquitto:

```bash
sudo systemctl enable mosquitto.service
```

### HARDWARIO Gateway Autostart

Create service:

```bash
sudo nano /etc/systemd/system/bcg.service
```

Example service:

```ini
[Unit]
Description=HARDWARIO BCG Gateway
After=network.target mosquitto.service

[Service]
Type=simple
User=iris
WorkingDirectory=/home/iris
ExecStart=/home/iris/bcg-env/bin/bcg --device /dev/ttyUSB0
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Enable:

```bash
sudo systemctl daemon-reload
sudo systemctl enable bcg.service
sudo systemctl start bcg.service
```

Verify:

```bash
systemctl status bcg.service
```

## Reboot Recovery

After power loss or reboot:

```text
Raspberry Pi boots
-> Mosquitto starts
-> Node-RED starts
-> BCG gateway starts
-> HARDWARIO reconnects
-> Node-RED restores local queue
-> Events continue uploading automatically
```

No manual intervention required.

## Offline Detection

Node-RED tracks:

```text
lastSeen timestamp
```

If no telemetry is received for:

```text
120 seconds
```

The node becomes:

```text
offline
```

## Downsampling

Temperature telemetry is buffered and averaged before upload.

Purpose:

- Reduce backend traffic
- Reduce MongoDB writes
- Reduce dashboard noise

Example:

```text
20 samples/minute
-> averaged
-> 1 upload/minute
```
